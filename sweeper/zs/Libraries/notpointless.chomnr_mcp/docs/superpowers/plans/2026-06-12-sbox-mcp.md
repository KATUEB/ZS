# s&box MCP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** An s&box library that hosts a Streamable-HTTP MCP server inside the editor with ~60 tools (scene, assets, ModelDoc, AnimGraph, ShaderGraph, ActionGraph, code, diagnostics) and a colorful dock UI with copy-paste client configs.

**Architecture:** All code in `Editor/` (unsandboxed editor assembly). `HttpListener` on 127.0.0.1 accepts MCP JSON-RPC over POST; tool calls hop to the editor main thread via a frame-pumped dispatcher. Tools are static methods discovered via `[McpTool]` reflection with auto-generated JSON Schema. A `[Dock]` widget renders status, configs, activity feed, tool browser, settings.

**Tech Stack:** C# (.NET as required by s&box, verify via dotnet build), System.Text.Json, s&box editor API (Sandbox.Tools/Sandbox.Engine from `D:\SteamLibrary\steamapps\common\sbox\bin\managed`), xunit for protocol-layer tests.

**Verification gates (every task):**
- `dotnet build dev/SboxMcp.Dev.csproj` → 0 errors (compiles all Editor/ sources against real s&box DLLs)
- `dotnet test dev/SboxMcp.Tests/SboxMcp.Tests.csproj` where tests exist
- `git commit` at end of each task

**Layering rule:** files under `Editor/Server/` and `Editor/Registry/` must NOT reference Sandbox/Editor namespaces (pure .NET) so the test project can compile them standalone. Sandbox-touching glue lives in `Editor/Integration/` and `Editor/Tools/`, `Editor/UI/`.

```csharp
public enum ToolCategory { Scene, GameObject, Component, Prefab, Asset, ModelDoc, AnimGraph, ShaderGraph, ActionGraph, Code, Editor }

[AttributeUsage(AttributeTargets.Method)]
public sealed class McpToolAttribute : Attribute {
    public string Name { get; }            // e.g. "gameobject_create"
    public string Description { get; }
    public ToolCategory Category { get; }
    public bool Writes { get; init; }
    public McpToolAttribute(string name, string description, ToolCategory category) {...}
}

[AttributeUsage(AttributeTargets.Parameter)]
public sealed class DescAttribute : Attribute { public string Text { get; } ... }

public sealed class RegisteredTool {
    public McpToolAttribute Meta; public MethodInfo Method;
    public McpToolDescriptor Descriptor;                  // schema from SchemaGenerator
    public object Invoke(JsonElement? args);              // binds args by param name w/ JsonSerializer, applies defaults, throws ToolArgumentException on missing required

public sealed class ToolRegistry {
    public void AddAssembly(Assembly asm);                // finds static methods with [McpTool]
    public IReadOnlyList<RegisteredTool> Tools { get; }
    public RegisteredTool Find(string name);
}
```

Optional params (HasDefaultValue) excluded from `required`. Tool return values: `string` passed through; other objects serialized to indented JSON.

### Task 4: McpServer — Streamable HTTP endpoint (pure .NET)

**Files:**
- Create: `Editor/Server/McpServer.cs`, `Editor/Server/McpSession.cs`
- Test: `dev/SboxMcp.Tests/ServerTests.cs` (real HttpListener on ephemeral port + HttpClient)

```csharp
public sealed class McpServer : IDisposable {
    public McpServer(ToolRegistry registry,
        Func<RegisteredTool, JsonElement?, Task<object>> invoker,  // injected: main-thread hop + permissions
        Action<ActivityRecord> onActivity = null);
    public int Port { get; private set; }
    public string Path { get; } = "/sbox-mcp";
    public bool IsRunning { get; }
    public IReadOnlyCollection<McpSession> Sessions { get; }
    public event Action StateChanged;
    public void Start(int port);    // throws HttpListenerException on busy port
    public void Stop();
}
public sealed class McpSession {
    public string Id;               // Guid("N")
    public string ClientName;       // from initialize clientInfo
    public DateTime ConnectedAt; public DateTime LastSeen; public int CallCount;
}
```

Request handling (background accept loop, `Task.Run` per request):
- `POST {Path}`: parse body. Methods: `initialize` (create session, emit `Mcp-Session-Id` response header, negotiate version), `notifications/initialized` (+ any `notifications/*`) → 202 empty body, `ping` → `{}`, `tools/list`, `tools/call` (`{name, arguments}` → invoker; exceptions → `TextContent(msg, isError:true)` result, NOT JSON-RPC error), unknown → MethodNotFound. Response `Content-Type: application/json`.
- `GET` → 405. `DELETE` → remove session, 200. Unknown path → 404.
- Reject non-localhost `Origin` headers (403) when present and not `localhost`/`127.0.0.1`/null.
- All exceptions per-request caught → 500 JSON-RPC InternalError; listener loop never dies; `Stop()` idempotent, closes listener + drains.

**Steps:**
- [ ] Failing tests with stub registry + echo invoker: initialize handshake (version + session header), tools/list roundtrip, tools/call success + isError on throwing invoker, notification → 202, GET → 405, bad JSON → -32700, stop/start cycle on same port.
- [ ] `dotnet test` FAIL → implement → PASS; Dev build PASS
- [ ] Commit `feat: Streamable HTTP MCP server`
